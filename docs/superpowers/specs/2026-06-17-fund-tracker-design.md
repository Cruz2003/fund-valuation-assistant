# 基金实时估值助手（Fund Tracker）设计规格

> 日期：2026-06-17 | 状态：已确认

## 一、项目概述

桌面版基金实时估值助手。用户输入基金代码或名称搜索添加基金，程序自动获取持仓和A股/港股/美股三市场实时行情，实时估算基金净值涨跌，分析每只股票对涨跌的贡献度。

## 二、技术选型

| 层面 | 技术 | 原因 |
|------|------|------|
| 语言 | Python 3.12 | spec要求 |
| GUI | PySide6 | 免费、功能强、可打包EXE |
| 数据 | Pandas + NumPy | 数据处理和计算 |
| 存储 | SQLite | 免安装、本地化 |
| 行情 | AkShare | A股/港股/美股数据 |
| 图表 | Matplotlib | 贡献度图、持仓饼图 |
| 其他 | requests, openpyxl | HTTP请求、Excel导出 |

## 三、架构设计

### 3.1 三层架构

```
GUI层 (gui/)
  ├── main_window.py       — 主窗口（左侧列表+右侧详情）
  ├── fund_list_panel.py   — 基金列表面板（搜索、添加、删除）

  └── dialogs.py           — 搜索对话框、设置对话框

  ├── fund_manager.py      — 基金管理（CRUD）
  ├── valuation_engine.py  — 估值计算引擎
  ├── data_fetcher.py      — 数据获取（AkShare封装）
  └── alert_service.py     — 阈值提醒服务

  ├── database.py          — SQLite数据库操作
  ├── models.py            — 数据模型定义

  ├── main.py              — 程序入口
  ├── config.py            — 配置管理

```

### 3.2 数据流

```
用户输入 → 搜索基金 → 确认添加 → SQLite存储
                                      ↓
定时/手动触发 ← scheduler ← 交易时段判断
      ↓
AkShare获取三市场行情 → 估值引擎计算
      ↓                        ↓
界面更新显示 ← 检查阈值 ← 持久化快照
```

## 四、功能设计

### 4.1 基金搜索与添加

- 输入方式：基金代码（6位）或关键词搜索
- 搜索结果展示：基金代码、名称、类型
- 确认后添加到列表，立即获取持仓数据

### 4.2 实时估值

**计算公式：**
```
实时估值 = 昨日净值 × (1 + Σ(持仓占比_i × 股票涨跌幅_i))
贡献度_i = 持仓占比_i × 股票涨跌幅_i × 100%
预计涨幅 = (实时估值 - 昨日净值) / 昨日净值 × 100%
```

**数据获取策略：**
- 首次添加：获取十大重仓股 + 持仓占比
- 定时刷新：每5分钟获取所有持仓股票实时行情
- 交易时段判断：不同市场交易时间不同，非交易时段跳过该市场
- 手动刷新：随时可触发

### 4.4 提醒机制

- 全局默认阈值 ±2%（可调整）
- 超过阈值：基金列表项高亮（红色涨/绿色跌）
- 不触发系统通知

### 4.5 数据持久化

SQLite 表结构：

```sql
-- 用户添加的基金
funds (id, code, name, fund_type, nav_yesterday, added_at)

-- 持仓快照（每次刷新更新）
holdings (id, fund_id, stock_code, stock_name, market, weight, updated_at)

-- 估值历史（每次刷新记录）
valuation_log (id, fund_id, estimated_nav, change_pct, created_at)

-- 用户配置
settings (key, value)
```

## 五、数据库设计

| 列名 | 类型 | 说明 |
|------|------|------|

## 七、注意事项

- AkShare 接口可能有频率限制，定时刷新间隔不宜过短（≥5分钟）
- 十大重仓股可能不是基金全部持仓，估值存在偏差
- 美股行情在晚上交易，需要北京时间晚间刷新才有效
- 多市场同时刷新时需处理网络异常，单个市场失败不影响其他
- 打包EXE时需要包含依赖和资源文件
